About this report

What this shows. Cumulative Flow Diagram: cumulative arrivals on top, cumulative departures on bottom, intermediate workflow states stacked between. Vertical distance at any sample date = WIP in that band; slope = average arrival rate. Past data only — no projections.

How to read the chart.

  • Each column is a snapshot taken on that date — every band's height is the count of items currently in that workflow step at that moment. Slide horizontally and you're watching the same set of pipelines change over time.
  • The top band is items currently in the first workflow step (Open for this report). That's where new arrivals land — items enter the system into this band. Anything not yet in the workflow (e.g., a Jira ticket that hasn't been filed; a PR not yet opened) is not on this chart.
  • The bottom band is items that have reached the final step (Merged) — finished, accumulating forever.
  • The total height at any date = cumulative items that have ever entered the system (top line, including the finished ones at the bottom).
  • The vertical gap between the top edge and the bottom edge = items still in flight (WIP). A widening gap means arrivals are outrunning departures; narrowing means the opposite.
  • The steeper a band's edge, the faster items are flowing into that step at that moment.
  • The horizontal gap between two adjacent lines at the same height tells you roughly how long items spent in the stages between them — a wide gap is a slow pipeline.

Key insight. Workflow has a single state — CFD collapses to one line of cumulative arrivals.

Next actions

  1. Look for places where one line flattens — that's a stall in that band.
  2. Compare band widths to the previous window to spot accumulation trends.

Caveats

  • Past data only — the chart does not project forward.
  • Items still in flight may be under-represented when the source only fetches completed work. Wider bands today may grow tomorrow.
  • Workflow used: Open → Merged. Items that visit statuses outside this list are silently skipped at those points.

Vocabulary

Cumulative Flow Diagram
Stacked area chart of cumulative counts per workflow state over time. Each state's line counts items that have entered that state or any later one — so the lines never cross and never decrease.
Arrivals
Top line of a CFD. Cumulative count of items that have entered the workflow (first state) by each sample date. Slope = arrival rate.
Departures
Bottom line of a CFD. Cumulative count of items that have exited the workflow (last state) by each sample date. Slope = throughput.
WIP
Work In Progress. Vertical distance between two adjacent CFD lines at a sample date = items currently in that workflow band. Per (see DECISIONS.md for the math).
Workflow state
A named stage in your delivery process (e.g., Open, In Progress, Done). CFDs require an ordered list — earliest stage to latest.
