About the GitHub samples: the default GitHub CFD uses Open → Merged because PRs don't expose a multi-state workflow out of the box. Teams that label their PRs (e.g. S-waiting-on-review, S-waiting-on-author) can drive a richer multi-band CFD by passing --wip-labels to flow aging; the same labeling pattern works for CFD when you pass an ordered --workflow. The GitHub Aging samples here use the simple review-decision lifecycle (Draft → Awaiting Review → Changes Requested → Approved); see docs/DECISIONS.md #9 and #10 for the full reasoning.
